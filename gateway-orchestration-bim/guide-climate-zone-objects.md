## Prerequisites

- Write access to the BIM Object vault (`woodfine-bim-library` or equivalent).
- Git configured with your commit identity.
- `ifctester` installed for IDS validation:
  ```bash
  pip install ifctester
  ```

## Purpose

This guide covers how Climate Zone performance data is structured in the BIM Object vault
and how it is consumed by `app-orchestration-bim`. Climate Zone BIM Objects store the
performance requirements that apply in each registered zone for a given element type.

## What Climate Zone Data Is

Climate Zone data is performance specification data embedded in BIM Objects. It is not a
user-selectable dropdown — it is reference data showing the performance requirements that
apply in each registered climate zone for a given element type.

The data lives in `tokens/bim/climate-zones.dtcg.json` in the BIM Object vault.

## File Location

```text
<bim-object-vault>/
└── tokens/
    └── bim/
        └── climate-zones.dtcg.json
```

Each entry in the file is a performance parameter for one element type in one climate zone:

```json
{
  "$type": "bim-climate-zone",
  "zone_id": "ASHRAE-5C",
  "ifc_class": "IfcWall",
  "parameter": "max_u_value",
  "required_value": 0.104,
  "unit": "Btu/h·ft²·°F",
  "source_standard": "ASHRAE 90.1-2022 Table 5.5-5",
  "$value": "climate-zone-spec"
}
```

## BIM Object Resolution Workflow

1. **Site assignment.** The project site is linked to an `IfcSpatialZone`
   (PredefinedType: `USERDEFINED`, ObjectType: `CLIMATE_ZONE`) with the applicable zone
   identifier (e.g., `ASHRAE-5C`).

2. **BIM Object lookup.** When a BIM Object is placed, the object resolver reads
   `climate-zones.dtcg.json` and finds all rows where `ifc_class` matches the placed
   element type and `zone_id` matches the project's assigned zone.

3. **Performance binding.** The matched rows are bound to the element as required property
   values. These appear in the element's Regulation tab as Climate Zone rows.

4. **IFC projection.** The resolved climate zone requirements are projected into the
   IFC-SPF archive as `IfcConstraint` records linked to the element via
   `IfcRelAssociatesConstraint`.

## Validation

Use `ifctester` with an IDS 1.0 contract generated from the climate zone data to verify
every element satisfies the climate zone requirements for its type:

```bash
# From the app-orchestration-bim project root
ifctester --ids climate-zones-bc-5c.ids model.ifc --report json
```

The IDS contract file for a climate zone is generated by the BIM Object vault tooling
(planned for v0.0.3). At v0.0.1, the IDS contract must be authored manually from the
`climate-zones.dtcg.json` parameter rows.

## Adding a New Climate Zone

To add performance data for a new climate zone:

1. Open `tokens/bim/climate-zones.dtcg.json` in the BIM Object vault.
2. Add rows for each element type and performance parameter in the new zone. Use the
   existing row structure.
3. Commit to the BIM Object vault repository:
   ```bash
   git commit -m "feat(climate-zones): add <zone-id> performance parameters"
   ```
4. Restart `app-orchestration-bim` on the deployment host to load new climate zone data.
5. If the zone requires an IDS constraint file, author the IDS file and add it to
   `climate/<zone-id>/` in the vault.

## Registered Climate Zones at v0.0.1

| Zone ID | Description | Element types covered |
|---|---|---|
| ASHRAE-5C | Marine (Pacific Northwest, BC coast) | IfcWall, IfcSlab, IfcWindow |
| ASHRAE-6 | Cold | IfcWall, IfcSlab, IfcWindow |
| ASHRAE-7A | Very cold | IfcWall, IfcSlab |
| NBC-Zone-6 | Canadian cold climate | IfcWall, IfcSlab, IfcRoof |

Additional zones are planned for v0.0.3 as part of the BC RS-1 regulatory overlay set.

## Expected Outcome

After adding a new climate zone entry and restarting `app-orchestration-bim`:

- The new zone appears in the Climate Zone tab for the relevant BIM Object at
  `https://bim.woodfinegroup.com/tokens`.
- `GET /api/climate-zones` returns the new zone ID in its response.

## Verification

```bash
# Confirm zone appears in API response
curl -s https://bim.woodfinegroup.com/api/climate-zones | jq 'keys'
# Expected: includes the new zone ID

# Confirm IDS validation passes for a model in the new zone
ifctester --ids climate-zones-<zone-id>.ids model.ifc --report json \
    | python3 -c "import sys,json; r=json.load(sys.stdin); print(r.get('status'))"
# Expected: pass
```
